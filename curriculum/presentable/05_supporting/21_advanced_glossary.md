# Advanced Glossary

**Level**: Advanced
**Scope**: Multi-agent, safety, production deployment

---

## Terms

1. **Policy engine**: Rules that gate actions based on risk or compliance. (Related: [Chapter 1: Safety Guardrails](../03_advanced/chapter_01_safety_guardrails.md))
2. **Human-in-the-loop (HITL)**: A required human approval step before action. (Related: [Chapter 1: Safety Guardrails](../03_advanced/chapter_01_safety_guardrails.md))
3. **Escalation policy**: Rules for when to transfer to a human operator. (Related: [Chapter 1: Safety Guardrails](../03_advanced/chapter_01_safety_guardrails.md))
4. **Model routing**: Selecting a model based on cost, latency, or risk. (Related: [Chapter 3: Production Deployment](../03_advanced/chapter_03_production_deployment.md))
5. **Guardrail**: A constraint that prevents unsafe behavior. (Related: [Chapter 1: Safety Guardrails](../03_advanced/chapter_01_safety_guardrails.md))
6. **Audit trail**: Immutable record of decisions and tool calls. (Related: [Chapter 5: Monitoring and Alerting](../03_advanced/chapter_05_monitoring_alerting.md))
7. **Canary deployment**: Gradual rollout to a small segment. (Related: [Chapter 3: Production Deployment](../03_advanced/chapter_03_production_deployment.md))
8. **Rollback**: Automatic revert after failure detection. (Related: [Chapter 3: Production Deployment](../03_advanced/chapter_03_production_deployment.md))
9. **SLO**: Service level objective for reliability or latency. (Related: [Chapter 5: Monitoring and Alerting](../03_advanced/chapter_05_monitoring_alerting.md))
10. **Runbook**: Step-by-step incident response instructions. (Related: [Chapter 5: Monitoring and Alerting](../03_advanced/chapter_05_monitoring_alerting.md))
11. **Error budget**: Allowed failure rate over time. (Related: [Chapter 5: Monitoring and Alerting](../03_advanced/chapter_05_monitoring_alerting.md))
12. **System prompt control**: Centralized management of system instructions. (Related: [Chapter 1: Safety Guardrails](../03_advanced/chapter_01_safety_guardrails.md))
13. **Prompt versioning**: Tracking prompt changes like code releases. (Related: [Chapter 3: Production Deployment](../03_advanced/chapter_03_production_deployment.md))
14. **Multi-agent coordination**: Orchestrating specialized agents. (Related: [Chapter 2: Multi-Agent Systems](../03_advanced/chapter_02_multi_agent_systems.md))
15. **Task decomposition**: Splitting a goal into sub-tasks for agents. (Related: [Chapter 2: Multi-Agent Systems](../03_advanced/chapter_02_multi_agent_systems.md))
16. **Consensus check**: Agreement between multiple agents before action. (Related: [Chapter 2: Multi-Agent Systems](../03_advanced/chapter_02_multi_agent_systems.md))
17. **Risk gating**: Blocking actions above a risk threshold. (Related: [Chapter 1: Safety Guardrails](../03_advanced/chapter_01_safety_guardrails.md))
18. **Data boundary**: Defined limits for data access or retention. (Related: [Chapter 6: Security Best Practices](../03_advanced/chapter_06_security_best_practices.md))
19. **Token cost tracking**: Measuring cost per request or task. (Related: [Chapter 4: Scaling Strategies](../03_advanced/chapter_04_scaling_strategies.md))
20. **Latency budget**: Maximum allowed response time. (Related: [Chapter 4: Scaling Strategies](../03_advanced/chapter_04_scaling_strategies.md))
21. **Blast radius**: The scope of impact if an agent fails. (Related: [Chapter 3: Production Deployment](../03_advanced/chapter_03_production_deployment.md))
22. **Operational telemetry**: Logs, traces, and metrics for live systems. (Related: [Chapter 5: Monitoring and Alerting](../03_advanced/chapter_05_monitoring_alerting.md))
